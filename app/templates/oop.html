Object Orientated Programming

With examples from the calculator

Encapsulation:

Making certain attributes and behaviors of an object private from other objects.
In the Calculator app, we encapsulated the methods to perform math operations.

Inheritance:

Creating objects (children) that take, or inherit, attributes and behaviors from their parent object
In the Calculator app, each of the math operations inherited behavior from a Calculation object

Polymorphism:

Defining an object that creates an outline for specific behaviors
In the Calculator app, polymorphism was used when we created the Calculation object which was used as an outline for the 4 math operations. The important thing to realize here is that all the math operations perform a calculation, but in a different way.

Abstraction:

Taking all of the behaviors and attributes of an object and condensing it into a format that is usable
In the Calculator app, the user should not be able to directly change the history. However, when they perform a math operation, the data of that operation will be automatically added to the history
